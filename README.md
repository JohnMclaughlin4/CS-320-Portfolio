# CS-320-Portfolio
Reflection
How can I ensure that my code, program, or software is functional and secure?

I can ensure my code is functional by writing unit tests, testing different input values, and checking that the program meets each requirement. In Project One, I used JUnit tests to verify that contacts could be created, updated, and deleted correctly. I also tested invalid inputs, such as null values or values that were too long, to make sure the program handled errors properly. To keep software secure, I should validate user input, limit what users are allowed to change, and avoid allowing invalid or unexpected data into the program.

How do I interpret user needs and incorporate them into a program?

I interpret user needs by carefully reviewing the requirements and turning them into specific program features. For example, the contact service required contacts to have a unique ID, limited field lengths, and the ability to update certain fields. I used those requirements to guide the design of the Contact and ContactService classes. This helped make sure the final program matched what the user needed instead of adding features that were not required.

How do I approach designing software?

I approach software design by breaking the problem into smaller parts. First, I look at the requirements and identify the main objects or services the program needs. Then I design classes with clear responsibilities. In this project, the Contact class stored contact information, while the ContactService class managed adding, deleting, and updating contacts. I also use testing throughout the process so I can catch problems early and confirm that each part works before moving on.
